# Implementation Command

タスクリストに基づいて実装を実行します。

## Instructions

以下の手順で実装を進めてください：

1. **タスクリストの確認**
   - `specs/<feature>/tasks.md` を読み込む
   - タスクリストがない場合は `/speckit.tasks` を先に実行するよう案内

2. **仕様書と計画の確認**
   - `specs/<feature>/spec.md` で要件を確認
   - `specs/<feature>/plan.md` で設計を確認
   - `memory/constitution.md` で原則を確認

3. **実装の実行**
   - タスクを順番に実行
   - 各タスク完了時にタスクリストを更新
   - 問題が発生した場合は報告

4. **品質確認**
   - コードがビルドできることを確認
   - テストが通ることを確認
   - 仕様との整合性を確認

## Implementation Guidelines

### コーディング原則

1. **Constitution遵守**
   - シンプルさを優先
   - 適切なエラーハンドリング
   - 一貫した命名規則

2. **変更の最小化**
   - 必要な変更のみ行う
   - 既存コードを尊重
   - 過剰な抽象化を避ける

3. **テスト**
   - 新機能にはテストを追加
   - 既存テストを壊さない

### 進捗報告

各タスク完了時に以下を報告：
- 完了したタスクID
- 変更したファイル
- 次のタスク

## Workflow

```
1. タスクリストを確認
     ↓
2. 次の未完了タスクを選択
     ↓
3. 関連ファイルを読み込む
     ↓
4. 実装を行う
     ↓
5. ビルド確認
     ↓
6. タスクリストを更新
     ↓
7. 次のタスクへ（または完了報告）
```

## Completion Checklist

実装完了時に以下を確認：

- [ ] すべてのタスクが完了
- [ ] ビルドが成功
- [ ] テストがパス
- [ ] 仕様書の要件を満たしている
- [ ] Constitutionの原則に準拠
- [ ] コードレビュー準備完了

## Output

実装の進捗を報告し、完了時は以下を含めてください：
- 完了したタスクの一覧
- 作成/変更したファイルの一覧
- 次のアクション（PRの作成など）
