# Feature Specification Command

新機能の仕様書を作成します。

## Input

機能名または説明: $ARGUMENTS

## Instructions

以下の手順で仕様書を作成してください：

1. **要件の理解**
   - ユーザーが説明した機能を理解する
   - 不明点があれば質問する

2. **仕様書の作成**
   - `specs/<feature-name>/spec.md` に仕様書を作成
   - 以下のセクションを含める：
     - 概要（Summary）
     - ユーザーストーリー（User Stories）
     - 機能要件（Functional Requirements）
     - 非機能要件（Non-Functional Requirements）
     - 成功基準（Success Criteria）
     - エッジケース（Edge Cases）

3. **Constitution確認**
   - `memory/constitution.md` の原則に違反していないか確認
   - 違反がある場合は指摘し、代替案を提案

4. **出力**
   - 仕様書のパスを報告
   - 次のステップ（/speckit.plan）を案内

## Template

```markdown
# [機能名] 仕様書

## 概要

[機能の目的と概要を1-2段落で説明]

## ユーザーストーリー

### US-001: [ストーリー名]
**As a** [ユーザー種別]
**I want to** [実現したいこと]
**So that** [得られる価値]

**受け入れ条件:**
- [ ] [条件1]
- [ ] [条件2]

## 機能要件

### FR-001: [要件名]
[要件の詳細説明]

## 非機能要件

### NFR-001: [要件名]
[パフォーマンス、セキュリティなどの要件]

## 成功基準

- [ ] [測定可能な基準1]
- [ ] [測定可能な基準2]

## エッジケース

| ケース | 期待される動作 |
|--------|---------------|
| [ケース1] | [動作1] |

## 技術的考慮事項

[実装時に考慮すべき技術的な点]
```

## Output

仕様書を `specs/` ディレクトリに保存し、作成完了を報告してください。
