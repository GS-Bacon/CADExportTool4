# Task Generation Command

実装計画からタスクリストを生成します。

## Instructions

以下の手順でタスクリストを生成してください：

1. **計画の確認**
   - `specs/<feature>/plan.md` を読み込む
   - 計画がない場合は `/speckit.plan` を先に実行するよう案内

2. **タスク分解**
   - ユーザーストーリーごとにタスクを分解
   - 各タスクは独立して実行・テスト可能であること
   - 並列実行可能なタスクには `[P]` マークを付与

3. **依存関係の整理**
   - 基盤タスク → コア機能タスク → 統合タスクの順序
   - 依存関係を明確に記載

4. **タスクリスト作成**
   - `specs/<feature>/tasks.md` に保存

## Task Format

各タスクは以下の形式で記載：

```
- [ ] [TaskID] [P?] [Story?] 説明 - ファイルパス
```

- **TaskID**: T001, T002, ... の形式
- **[P]**: 並列実行可能な場合に付与
- **[Story]**: 関連するユーザーストーリー（US-001など）

## Tasks Template

```markdown
# [機能名] タスクリスト

## 概要

**仕様書**: specs/<feature>/spec.md
**計画書**: specs/<feature>/plan.md
**総タスク数**: X
**並列実行可能**: Y

## セットアップ

- [ ] T001 [P] feature/<feature-name> ブランチを作成

## 基盤タスク

> すべてのユーザーストーリーの前提となるタスク

- [ ] T002 [基盤] [タスク説明] - [ファイルパス]

## ユーザーストーリー別タスク

### US-001: [ストーリー名]

- [ ] T003 [US-001] [タスク説明] - [ファイルパス]
- [ ] T004 [P] [US-001] [タスク説明] - [ファイルパス]

### US-002: [ストーリー名]

- [ ] T005 [US-002] [タスク説明] - [ファイルパス]

## テストタスク

- [ ] T010 [テスト] 単体テストを作成 - [テストファイルパス]
- [ ] T011 [テスト] 統合テストを作成 - [テストファイルパス]

## 仕上げタスク

- [ ] T012 [仕上げ] コードレビュー対応
- [ ] T013 [仕上げ] ドキュメント更新

## 実行順序

```
T001 (セットアップ)
  ↓
T002 (基盤)
  ↓
T003 → T004 (US-001, 並列可)
  ↓
T005 (US-002)
  ↓
T010 → T011 (テスト)
  ↓
T012 → T013 (仕上げ)
```

## 進捗

| フェーズ | 完了 | 合計 | 進捗率 |
|---------|------|------|--------|
| セットアップ | 0 | 1 | 0% |
| 基盤 | 0 | X | 0% |
| 機能実装 | 0 | Y | 0% |
| テスト | 0 | Z | 0% |
| 仕上げ | 0 | 2 | 0% |
```

## Output

タスクリストを作成し、次のステップ（/speckit.implement）を案内してください。
